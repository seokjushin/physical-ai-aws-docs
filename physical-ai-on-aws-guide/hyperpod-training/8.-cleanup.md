# 6. 리소스 정리

실습이 끝나면 CDK로 전체 인프라를 삭제합니다. 삭제하지 않으면 Head Node와 FSx 비용이 계속 발생합니다.

{% hint style="info" %}
이 문서의 명령어는 `${USER_ID}` 환경변수를 사용합니다. 새 셸 세션이라면 먼저 다음을 실행하세요:
```bash
export USER_ID=<본인이름>   # 1단계에서 사용한 것과 동일한 값
```
{% endhint %}

---

### 6.1 실행 중인 작업 확인 및 취소

```bash
# Head Node에서 실행
squeue

# 실행 중인 작업이 있다면 모두 취소
scancel --user=ubuntu
```

---

### 6.2 CDK 스택 삭제 (~20분)

CloudShell 또는 로컬에서 실행합니다:

```bash
cd ~/aws-physical-ai-recipes/hyperpod-training/infra

npx cdk destroy -c userId=${USER_ID} -c region=us-east-1 --force
```

---

### 6.3 삭제 실패 시 조치

S3 버킷이 비어있지 않으면 스택 삭제가 실패할 수 있습니다:

```bash
# 버킷 비우기
aws s3 rm s3://hyperpod-lifecycle-hyperpod-${USER_ID}-<ACCOUNT_ID>-us-east-1 --recursive
aws s3 rm s3://hyperpod-data-hyperpod-${USER_ID}-<ACCOUNT_ID>-us-east-1 --recursive

# 스택 삭제 재시도
aws cloudformation delete-stack \
  --stack-name HyperPod-${USER_ID} \
  --region us-east-1
```

---

### 6.4 삭제 확인

```bash
aws cloudformation describe-stacks \
  --stack-name HyperPod-${USER_ID} \
  --region us-east-1 2>&1

# "Stack with id HyperPod-${USER_ID} does not exist" 가 출력되면 정상 삭제 완료
```
