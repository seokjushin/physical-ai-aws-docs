# 9. Edge Operation

이전 모듈에서 fine-tuning한 GR00T 모델이 시뮬레이션 환경에서 정상적으로 동작하는 것을 [모듈 8](8.-evaluation.md)에서 확인했습니다. 이 모듈에서는 그 모델을 실제 엣지 디바이스인 **NVIDIA Jetson AGX Thor**에 배포하고, Thor에 연결된 **SO-ARM 101** 로봇팔을 추론 결과로 제어하는 단계를 다룹니다. 클라우드에서 학습·평가된 모델을 엣지로 안전하게 전달하고, 디바이스 상태를 모니터링하며, 새로운 버전이 나왔을 때 OTA 업데이트할 수 있는 파이프라인을 AWS IoT 서비스로 구성합니다.

엣지 운영 단계에서는 단순히 모델을 디바이스에 복사하는 것 이상이 필요합니다. 어떤 Thor 디바이스에 어떤 버전의 모델이 배포되어 있는지 관리하고, 추론 결과·로그·텔레메트리를 클라우드로 전송해 운영 가시성을 확보해야 합니다. 또한 모델이 업데이트되었을 때 fleet 전체에 일관되게 롤아웃하고, 문제가 발생하면 롤백할 수 있어야 합니다. 이러한 요구사항을 **AWS IoT Greengrass**, **AWS IoT Core**, **S3** 등을 조합해 해결합니다.

| 구분 | [모듈 8](8.-evaluation.md) (Closed-loop 평가) | 모듈 9 (Edge Operation) |
|------|----------------------------------------------|--------------------------|
| 실행 환경 | Isaac Lab 시뮬레이션 (DCV GPU 인스턴스) | Jetson AGX Thor + SO-ARM 101 |
| 모델 위치 | EFS (클라우드) | Thor 로컬 스토리지 |
| 배포 방식 | Policy Server 컨테이너 직접 실행 | AWS IoT Greengrass component |
| 검증 목적 | 시뮬레이션 태스크 성공률 | 실물 로봇 제어 동작 확인 |

본 워크숍에서는 [NVIDIA Jetson AGX Thor](https://www.nvidia.com/en-us/autonomous-machines/embedded-systems/jetson-thor/)를 엣지 추론 디바이스로 사용합니다. Thor는 Blackwell 기반 GPU가 탑재된 차세대 로보틱스 SoC로, 온디바이스에서 GR00T와 같은 대규모 VLA 모델을 실시간으로 추론할 수 있는 성능을 제공합니다. Thor는 USB/Serial을 통해 [HuggingFace LeRobot](https://github.com/huggingface/lerobot) 호환 오픈소스 로봇팔인 **SO-ARM 101**(6 DOF + gripper)에 연결되며, 모델이 출력한 action을 직접 로봇팔에 전달해 태스크를 수행합니다.
