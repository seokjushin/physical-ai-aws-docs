# 2. EKS 클러스터 설정

CDK 배포가 완료된 후, Kubernetes 클러스터에 접근하고 GPU 워크로드 실행에 필요한 구성요소를 설치합니다.

**설정 순서가 중요합니다:**

1. kubeconfig → 모든 kubectl 명령의 전제 조건
2. NVIDIA Device Plugin → GPU 리소스 인식 인프라
3. StorageClass / Namespace → 워크로드 실행 환경 준비
4. Cluster Autoscaler → GPU 노드 자동 확장 활성화

---

### 2.1 kubeconfig 설정

kubeconfig는 kubectl이 EKS 클러스터와 통신하기 위한 인증 정보 파일입니다. 이 명령을 실행하면 로컬 머신에서 클러스터를 제어할 수 있게 됩니다.

```bash
aws eks update-kubeconfig --name osmo-eks --region us-east-1
```

접속 확인:

```bash
kubectl get nodes
```

**정상 출력:**

```
NAME                                       STATUS   ROLES    AGE   VERSION
ip-10-0-1-xxx.us-east-1.compute.internal   Ready    <none>   5m    v1.30.x
ip-10-0-3-xxx.us-east-1.compute.internal   Ready    <none>   5m    v1.30.x
```

{% hint style="info" %}
System 노드 2대만 보이는 것이 정상입니다. GPU 노드는 워크플로 제출 시 Cluster Autoscaler가 자동으로 프로비저닝합니다.
{% endhint %}

---

### 2.2 NVIDIA Device Plugin 설치

Kubernetes는 기본적으로 GPU를 인식하지 못합니다. NVIDIA Device Plugin은 각 노드에서 GPU를 탐지하여 Kubernetes 스케줄러가 `nvidia.com/gpu` 리소스를 할당할 수 있게 합니다. DaemonSet으로 배포되어 모든 노드에 자동 설치됩니다.

```bash
kubectl apply -f https://raw.githubusercontent.com/NVIDIA/k8s-device-plugin/v0.14.5/nvidia-device-plugin.yml
```

확인:

```bash
kubectl get daemonset -n kube-system nvidia-device-plugin-daemonset
```

---

### 2.3 gp3 StorageClass 생성

StorageClass는 Pod가 영구 볼륨(PVC)을 요청할 때 "어떤 종류의 디스크를 사용할지" 정의합니다. EKS 기본값인 gp2 대신 최신 gp3를 기본으로 설정하면 더 나은 성능(3,000 IOPS 기본 제공)과 비용 효율을 얻을 수 있습니다.

EBS CSI Driver를 사용하는 고성능 스토리지 클래스를 기본값으로 설정합니다:

```bash
kubectl apply -f - <<'EOF'
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: gp3
  annotations:
    storageclass.kubernetes.io/is-default-class: "true"
provisioner: ebs.csi.aws.com
parameters:
  type: gp3
volumeBindingMode: WaitForFirstConsumer
allowVolumeExpansion: true
EOF
```

확인:

```bash
kubectl get sc
```

**정상 출력:**

```
NAME            PROVISIONER             RECLAIMPOLICY   VOLUMEBINDINGMODE      AGE
gp2             kubernetes.io/aws-ebs   Delete          WaitForFirstConsumer   10m
gp3 (default)   ebs.csi.aws.com         Delete          WaitForFirstConsumer   5s
```

OSMO 워크로드가 실행될 네임스페이스를 생성합니다:

```bash

```

### 2.5 Cluster Autoscaler 설치

Cluster Autoscaler는 Pending Pod를 감지하여 GPU 노드를 자동으로 추가/제거합니다. 이를 통해 GPU 인스턴스를 항상 켜두지 않고, 워크플로가 제출될 때만 자동 프로비저닝하여 비용을 절감합니다.

{% hint style="warning" %}
설치 직후 Cluster Autoscaler Pod가 `CrashLoopBackOff` 상태가 될 수 있습니다. 이는 정상입니다 — AWS API를 호출할 권한(IRSA)이 아직 없기 때문입니다. [3. IRSA 설정](3.-irsa-setup.md)을 완료하면 정상 동작합니다.
{% endhint %}

{% hint style="info" %}
Cluster Autoscaler의 IRSA(IAM Role for Service Account)는 [3. IRSA 설정](3.-irsa-setup.md)에서 구성합니다. 여기서는 Helm 설치만 진행합니다.
{% endhint %}

```bash
helm repo add autoscaler https://kubernetes.github.io/autoscaler
helm repo update autoscaler

helm install cluster-autoscaler autoscaler/cluster-autoscaler \
  --namespace kube-system \
  --set autoDiscovery.clusterName=osmo-eks \
  --set awsRegion=us-east-1 \
  --set image.tag=v1.30.2 \
  --set rbac.serviceAccount.create=true \
  --set rbac.serviceAccount.name=cluster-autoscaler
```

**Helm 파라미터 설명:**

| 파라미터 | 의미 |
|----------|------|
| `autoDiscovery.clusterName` | CA가 이 클러스터의 ASG만 관리하도록 필터링 |
| `awsRegion` | ASG API를 호출할 리전 |
| `image.tag=v1.30.2` | EKS 버전과 일치하는 CA 버전 (필수) |
| `rbac.serviceAccount.name` | IRSA 연결 시 사용할 ServiceAccount 이름 |

{% hint style="info" %}
Cluster Autoscaler 버전은 EKS 버전과 맞춰야 합니다. EKS 1.30을 사용하므로 `v1.30.x`를 설치합니다.
{% endhint %}

확인:

```bash
kubectl get pods -n kube-system -l app.kubernetes.io/name=aws-cluster-autoscaler
```

**정상 출력:**

```
NAME                                                         READY   STATUS    AGE
cluster-autoscaler-aws-cluster-autoscaler-xxxxxxxx-xxxxx     1/1     Running   30s
```

### Troubleshooting

| 증상 | 원인 | 해결 |
|------|------|------|
| `error: You must be logged in` | kubeconfig 미설정 또는 권한 없음 | `aws eks update-kubeconfig` 재실행, IAM 사용자가 EKS Access Entry에 등록되었는지 확인 |
| CA Pod CrashLoopBackOff | IRSA 미설정 (AWS 자격 증명 없음) | [3. IRSA 설정](3.-irsa-setup.md) 완료 후 CA 재시작 |
| CA leader lease 획득 실패 | 이전 Pod의 stale lease | `kubectl delete lease cluster-autoscaler -n kube-system` |
| NVIDIA Device Plugin 미동작 | GPU 노드 미존재 | 정상. GPU 노드 scale-up 시 자동 실행됨 |
