# 6. 정리 (Cleanup)

실습이 완료되면 모든 리소스를 삭제하여 불필요한 비용 발생을 방지합니다. 아래 순서대로 진행합니다.

{% hint style="warning" %}
정리 순서가 중요합니다. Kubernetes 리소스 → IAM → CDK 스택 순서로 삭제해야 의존성 문제를 방지할 수 있습니다.
{% endhint %}

---

### 6.1 실행 중인 Workflow 중단

```bash
# 실행 중인 workflow 확인
osmo workflow list --status running

# 실행 중인 workflow 취소
osmo workflow cancel <workflow-id>

# 남아있는 Pod 확인 및 삭제
kubectl delete jobs --all -n osmo
kubectl get pods -n osmo
```

### 6.2 Kubernetes 리소스 정리

```bash
# Helm 릴리스 삭제
helm uninstall cluster-autoscaler -n kube-system

# NVIDIA Device Plugin 삭제

# osmo 네임스페이스 삭제 (내부 리소스 포함)
kubectl delete namespace osmo
```

### 6.3 GPU 노드 Scale-down 확인

CDK 스택 삭제 전에 GPU 노드가 0대인지 확인합니다:

```bash
aws autoscaling describe-auto-scaling-groups \
  --query 'AutoScalingGroups[?contains(AutoScalingGroupName, `eks-gpu`)].{Name:AutoScalingGroupName,Desired:DesiredCapacity}' \
  --output table
```

0이 아닌 경우 수동으로 축소:

```bash
# gpu-sim
aws autoscaling describe-auto-scaling-groups \
  --query 'AutoScalingGroups[?contains(AutoScalingGroupName, `gpu-sim`)].AutoScalingGroupName' \
  --output text | xargs -I{} aws autoscaling set-desired-capacity \
  --auto-scaling-group-name {} --desired-capacity 0

# gpu-train
aws autoscaling describe-auto-scaling-groups \
  --query 'AutoScalingGroups[?contains(AutoScalingGroupName, `gpu-train`)].AutoScalingGroupName' \
  --output text | xargs -I{} aws autoscaling set-desired-capacity \
  --auto-scaling-group-name {} --desired-capacity 0
```

### 6.4 IAM 리소스 삭제

```bash
ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)

# Inline policy 삭제
aws iam delete-role-policy --role-name osmo-workload-role --policy-name osmo-s3-access
aws iam delete-role-policy --role-name osmo-cluster-autoscaler-role --policy-name cluster-autoscaler-policy

# IAM Role 삭제
aws iam delete-role --role-name osmo-workload-role
aws iam delete-role --role-name osmo-cluster-autoscaler-role

# OIDC Provider 삭제
OIDC_URL=$(aws eks describe-cluster --name osmo-eks \
  --query 'cluster.identity.oidc.issuer' --output text 2>/dev/null)
if [ -n "$OIDC_URL" ]; then
  OIDC_PROVIDER_ARN=$(aws iam list-open-id-connect-providers \
    --query "OpenIDConnectProviderList[?ends_with(Arn, '$(echo $OIDC_URL | cut -d/ -f5)')].Arn" \
    --output text)
  if [ -n "$OIDC_PROVIDER_ARN" ]; then
    aws iam delete-open-id-connect-provider --open-id-connect-provider-arn "$OIDC_PROVIDER_ARN"
  fi
fi
```

### 6.5 CDK 스택 삭제 (~15분)

```bash
cd ~/aws-physical-ai-recipes/osmo/cdk
npx cdk destroy --force
```

삭제 진행 확인:

```bash
aws cloudformation describe-stacks --stack-name Osmo \
  --query 'Stacks[0].StackStatus' --output text
```

`DELETE_COMPLETE` 또는 "Stack does not exist"가 되면 완료입니다.

{% hint style="info" %}
CDK destroy는 약 15분 소요됩니다. EKS 클러스터와 RDS 삭제에 시간이 걸립니다.
{% endhint %}

### 6.6 S3 데이터 삭제 (선택)

{% hint style="warning" %}
`cdk destroy`는 S3 버킷의 데이터를 삭제하지 않습니다. 데이터를 보존하려면 이 단계를 건너뛰세요.
{% endhint %}

```bash
BUCKET=$(aws cloudformation describe-stacks --stack-name Osmo \
  --query 'Stacks[0].Outputs[?OutputKey==`S3BucketName`].OutputValue' --output text 2>/dev/null)

# 버킷이 존재하면 비우고 삭제
if [ -n "$BUCKET" ]; then
  aws s3 rm s3://$BUCKET --recursive
  aws s3 rb s3://$BUCKET
fi
```

### 6.7 삭제 확인

모든 리소스가 정상적으로 삭제되었는지 확인합니다:

```bash
# CloudFormation 스택 확인
aws cloudformation describe-stacks --stack-name Osmo 2>&1 | grep -q "does not exist" && echo "Stack deleted" || echo "Stack still exists"

# IAM Role 확인
aws iam get-role --role-name osmo-workload-role 2>&1 | grep -q "NoSuchEntity" && echo "Role deleted" || echo "Role exists"
aws iam get-role --role-name osmo-cluster-autoscaler-role 2>&1 | grep -q "NoSuchEntity" && echo "Role deleted" || echo "Role exists"

# EKS 클러스터 확인
aws eks describe-cluster --name osmo-eks 2>&1 | grep -q "ResourceNotFoundException" && echo "Cluster deleted" || echo "Cluster exists"
```

**모두 "deleted"로 출력되면 정리 완료입니다.**

### Troubleshooting

| 증상 | 원인 | 해결 |
|------|------|------|
| `DELETE_FAILED` | 노드그룹이 scale-down 중 | 6.3 단계에서 desired=0 설정 후 5분 대기, 재시도 |
| `Cannot delete role, has inline policies` | 정책 미삭제 | `aws iam delete-role-policy` 먼저 실행 |
| EKS 클러스터 삭제 hang | Node group 삭제 대기 | AWS 콘솔에서 EKS → Node Groups 상태 확인 |
| S3 버킷 삭제 실패 | 버킷이 비어있지 않음 | `aws s3 rm s3://BUCKET --recursive` 먼저 실행 |
| `FORCE_DELETE_STACK` 필요 | 리소스 삭제 실패 반복 | `aws cloudformation delete-stack --stack-name Osmo` 후 콘솔에서 "Retain" 옵션으로 강제 삭제 |
