# 0. NX1 환경 셀프 배포 가이드 (먼저 읽기)

이 워크샵은 **Doosan Bobcat BEST NX1** 계정에서 진행됩니다. NX1은 일반 AWS 계정과 거버넌스가 달라, 원본 가이드의 **CloudShell · `cdk deploy` · SSH · DCV 공인 URL · CloudFront** 방식이 모두 막혀 있습니다. 아래 NX1 방식으로 대체해 진행하세요. (강사가 제공하는 **템플릿 S3 URL**과 **스크립트 zip URL**을 미리 받아두세요.)

> 원본 모듈 1·5의 "CloudShell/CDK 배포", 모듈 1·4·8의 "DcvUrl 접속", 모듈 9의 "code-server(CloudFront)"는 **이 페이지의 방식으로 읽으세요.**

## ⚠️ NX1 핵심 차이 (먼저)
- **리소스 직접 생성 불가** → 배포는 **CloudFormation 콘솔**(서비스롤 `CloudFormationDeployer` + 태그 `doosan:owner=BEST_NX1`).
- **CloudShell 차단, SSH 차단** → CLI는 **SSM Session Manager**(브라우저 셸), GUI는 **SSM 포트포워딩**.
- **공유 VPC**(per-user VPC 생성 불가) → 본인 `UserId`로만 리소스 구분. **다른 사람과 겹치지 않는 UserId**를 쓰세요(소문자·숫자·하이픈).
- **한 번에 완료**: 인스턴스는 무활동 시 자동 정지되고 **본인은 다시 켤 수 없습니다**(강사만). 자리를 오래 비우지 마세요.

## 사전 준비 (로컬, 1회)
1. **AWS Identity Center 로그인** → 역할 `BESTNX1-Developer` (Doosan AD + MFA). 포털 URL은 강사 제공.
2. (DCV/code-server GUI용) 로컬에 **AWS CLI v2** + **Session Manager plugin** 설치 (Windows/Mac):
   - AWS CLI v2: https://aws.amazon.com/cli/
   - Session Manager plugin: https://docs.aws.amazon.com/systems-manager/latest/userguide/session-manager-working-with-install-plugin.html
3. 하루 1회: `aws sso login --profile <강사-제공-프로파일>`

## Step 1. 인프라 배포 (CloudFormation 콘솔 — 로컬 설치 0)
원본 모듈 1의 "CloudShell + `cdk deploy`" 대신:
1. 콘솔 → **CloudFormation** → **Create stack** → *With new resources*.
2. **Template source = Amazon S3 URL** → 강사가 준 **템플릿 S3 URL** 입력. *(로컬 "파일 업로드"는 권한 밖이라 실패합니다.)*
3. **Stack name**: `nx1-isaaclab-<본인UserId>` (예 `nx1-isaaclab-alice`).
4. **Parameters**:
   - `UserId` = 본인 ID(소문자/숫자/하이픈, 고유)
   - `InstanceType` = `g6.4xlarge`
   - `ScriptsZipUrl` = 강사 제공 zip URL
   - `EnableCodeServer` = `true` (VSCode 쓰려면), 나머지 기본값
5. **Configure stack options → Permissions → IAM role** = **`CloudFormationDeployer`** ⚠️필수
6. **Tags** → Key `doosan:owner`, Value `BEST_NX1` ⚠️필수(없으면 생성 거부)
7. **Submit** → `CREATE_IN_PROGRESS`. UserData 빌드로 **약 30–35분** 후 `CREATE_COMPLETE`.
   - 진행/오류는 스택 **Events** 탭.

## Step 2. CLI 핸즈온 (S3 / nvidia-smi / docker / EFS)
원본의 "SSH 접속" 대신:
- 콘솔 → **Systems Manager** → **Session Manager** → **Start session** → 대상 `nx1-isaaclab-<UserId>` → 브라우저 셸.
- 인스턴스 역할로 동작합니다:
  ```bash
  sudo su - ubuntu
  nvidia-smi
  df -h /home/ubuntu/environment/efs    # EFS 마운트 확인
  docker images
  ```

## Step 3. GUI 접속 — DCV / code-server (SSM 포트포워딩)
원본의 "DcvUrl"·"CodeServer(CloudFront) URL" 대신, **로컬 터미널**에서 SSM 포트포워딩으로 접속합니다.

> **`<INSTANCE_ID>` 란?** 본인 GPU 인스턴스의 EC2 ID(`i-…`). private 서브넷이라 공인 IP가 없어, 포트포워딩 `--target`에 이 ID로 지정합니다.

**인스턴스 ID 찾기 (가장 쉬운 순):**
1. **CloudFormation 콘솔 → 본인 스택 `nx1-isaaclab-<UserId>` → Outputs 탭** →
   - **`PortForwardCmd`** = instance ID가 이미 박힌 **완성 DCV 포트포워딩 명령** → 그대로 복사해 실행(아래 8443 명령 대체).
   - **`InstanceId`** = `i-…` 값.
2. 또는 CLI(태그 조회):
   ```bash
   aws ec2 describe-instances \
     --filters "Name=tag:Name,Values=nx1-isaaclab-<UserId>" "Name=instance-state-name,Values=running" \
     --query "Reservations[].Instances[].InstanceId" --output text \
     --region us-east-1 --profile <강사-제공-프로파일>
   ```

**DCV (Isaac Sim GUI)** — 포트 8443:
```bash
aws ssm start-session --target <INSTANCE_ID> \
  --document-name AWS-StartPortForwardingSession \
  --parameters portNumber=8443,localPortNumber=8443 \
  --region us-east-1 --profile <강사-제공-프로파일>
```
→ 브라우저 `https://localhost:8443` (자가서명 인증서 경고 통과).

**code-server (VSCode)** — 포트 8888:
```bash
aws ssm start-session --target <INSTANCE_ID> \
  --document-name AWS-StartPortForwardingSession \
  --parameters portNumber=8888,localPortNumber=8888 \
  --region us-east-1 --profile <강사-제공-프로파일>
```
→ 브라우저 `http://localhost:8888`.

- 로그인 비밀번호(DCV·code-server 공통): 콘솔 **Secrets Manager** → `nx1-isaaclab-<UserId>-secret` 값. 사용자명 `ubuntu`.
- reboot 직후 SSM 재연결에 1–2분 걸릴 수 있습니다("TargetNotConnected" 시 잠시 후 재시도).

## Day 2 참고 (모듈 5–8)
- **SageMaker Studio**(모듈 5·7): Identity Center 포털에서 SageMaker Studio 접속 → 로그인 시 본인 user-profile이 자동 생성됩니다. CDK로 도메인 생성하지 않습니다(공유 도메인 사용).
- **학습**(모듈 7): SageMaker Training Job은 **On-Demand**로 실행하세요(`--no-spot`). **VPC 미바인딩**(기본)으로 둡니다.
- **MLflow**(모듈 7): 학습 metric은 공유 MLflow에 자동 로깅됩니다. tracking server ARN은 강사 제공(또는 config에 사전 주입). UI는 Studio에서 확인.
- **평가**(모듈 8 closed-loop): IsaacSim GUI가 필요하므로 **DCV(Step 3)** 에서 실행합니다. SSH 터미널로는 렌더링이 안 됩니다.

## Step 4. 종료 / 정리
- **한 번에 완료** — 자리를 오래 비우면 인스턴스가 자동 정지되고 본인은 못 켭니다(강사 문의).
- 정리: 콘솔 → CloudFormation → 본인 스택 → **Delete**. (S3 버킷에 객체가 있으면 비운 뒤 삭제.)

## 자주 막히는 곳
| 증상 | 해결 |
|---|---|
| Create stack "Upload" 후 실패 | 템플릿을 **S3 URL**로 지정(업로드 X) |
| `CreateStack` 거부 | IAM role=`CloudFormationDeployer` + 태그 `doosan:owner=BEST_NX1` 확인 |
| Session Manager 대상에 인스턴스 없음 | 빌드 미완(`CREATE_COMPLETE` 전) 또는 SSM 에이전트 기동 대기 — 몇 분 후 |
| `localhost:8443`/`:8888` 안 됨 | 포트포워딩 세션 살아있는지, 로컬 포트 충돌 없는지 |
| 인스턴스 꺼져 있음 | 무활동 자동 정지 → 강사에게 재시작 요청 |
